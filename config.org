#+TITLE: NixOS Literate Configuration
#+PROPERTY: header-args:nix :comments no :mkdirp yes :noweb yes

This document is the canonical source for the full NixOS configuration.
In literate style, we first explain each concern, then weave named code chunks
into concrete files via Org Babel tangling.

Functionally, this machine profile is organized around five outcomes:
- a stable NixOS-on-WSL base,
- reproducible host composition,
- core development tooling,
- Copilot CLI availability,
- and an Emacs-first working environment.

* dependency graph and system entrypoint :FLAKE:

The flake has three responsibilities: declare repository intent, pin inputs, and
define outputs for the =nixos= host.

From a functional perspective, this section is the contract that turns this
repository into a reproducible workstation definition: same inputs, same module
graph, same host outcome.

** Flake description chunk

This short chunk states what this flake represents.

#+name: flake-description
#+begin_src nix :tangle no
description = "NixOS WSL configuration";
#+end_src

** Flake input chunks

We split each upstream input into its own chunk so each dependency can be read
and changed independently.

#+name: flake-input-nixpkgs
#+begin_src nix :tangle no

#+end_src

#+name: flake-input-nixos-wsl
#+begin_src nix :tangle no

  url = "github:nix-community/NixOS-WSL";
  inputs.nixpkgs.follows = "nixpkgs";
};
#+end_src

#+name: flake-input-copilot-cli
#+begin_src nix :tangle no
copilot-cli = {
  url = "github:scarisey/copilot-cli-flake";
  inputs.nixpkgs.follows = "nixpkgs";
};
#+end_src

The enclosing input set is woven from the individual input chunks.

#+name: flake-inputs
#+begin_src nix :tangle no
inputs = {
  <<flake-input-nixpkgs>>

  <<flake-input-nixos-wsl>>

  <<flake-input-copilot-cli>>
};
#+end_src

*** automatic rolling-style upgrades :FUNCTIONALITY:

Enable unattended system upgrades and prefer rolling package refreshes.

Per request, this machine should update itself automatically and track newer
package revisions over time. To preserve local host behavior while still
getting rolling-style package updates, auto-upgrade rebuilds from this local
flake path and refreshes the =nixpkgs= input during each upgrade run.

#+name: module-base-auto-upgrade
#+begin_src nix :tangle no
system.autoUpgrade = {
  enable = true;
  dates = "daily";
  randomizedDelaySec = "45min";
  flake = "/home/nixos/nixos";
  flags = [
    "--update-input"
    "nixpkgs"
  ];
  allowReboot = false;
};
#+end_src

*** state version pin :FUNCTIONALITY:

Pin stateVersion to preserve migration semantics across upgrades.
